# Labs

## Rôle

Les labs valident des hypothèses d'architecture par des expériences limitées, observables et reproductibles.

Ils ne sont pas des tutoriels. Ils ne remplacent pas les chapitres conceptuels.

## Format attendu

Chaque lab doit préciser :

- l'hypothèse testée ;
- le périmètre minimal de l'expérience ;
- les composants observés ;
- les métriques ou signaux collectés ;
- le résultat qui confirmerait, nuancerait ou invaliderait l'hypothèse ;
- la décision d'architecture rendue possible par l'expérience.

## Critères de validation

Un lab est valide si :

- il teste une hypothèse formulée avant l'expérience ;
- il peut échouer sans perdre son intérêt pédagogique ;
- il produit une observation exploitable ;
- il relie le résultat à un choix d'architecture ;
- il reste indépendant d'un outil remplaçable.

## Hors périmètre

- Installer pas à pas un outil.
- Promouvoir une solution particulière.
- Masquer les limites ou les résultats négatifs.
- Transformer un lab en chapitre conceptuel.
