# Interfaces, orchestration, observabilité

## Intention

Définir les trois piliers d'un système d'IA exploitable.

## Plan de recadrage

Ce chapitre doit rester un chapitre de fondation, pas un chapitre d'architecture complète.

Responsabilité principale :

- introduire les interfaces comme frontières entre composants ;
- introduire les contrats comme moyen de rendre ces frontières vérifiables ;
- montrer pourquoi un système sans interfaces explicites devient difficile à comprendre.

À traiter seulement comme ouverture :

- l'orchestration comme coordination de composants déjà séparés ;
- l'observabilité comme moyen de comprendre un comportement en production ;
- les liens futurs avec agents, mémoire, outils et feedback.

Hors périmètre :

- les patterns d'orchestration détaillés ;
- les architectures multi-agents ;
- les métriques d'évaluation complètes ;
- les pipelines d'observabilité applicatifs.

## Questions à traiter

- Qu'est-ce qu'une bonne interface entre modèle et outil ?
- Quel rôle joue l'orchestrateur ?
- Que faut-il observer pour comprendre une décision ?
