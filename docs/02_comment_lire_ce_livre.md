# Comment lire ce livre

## Intention

Guider les parcours de lecture selon les profils et les objectifs.

## Questions à traiter

- Quel parcours suivre pour débuter ?
- Quel parcours suivre pour architecturer un assistant local ?
- Quand lire les labs par rapport aux chapitres conceptuels ?

## Ordre recommandé

1. Lire `foundations/` pour poser les invariants système.
2. Lire `llm/` puis `engines/` pour comprendre modèle et exécution.
3. Lire `agents/` puis `memory/` pour introduire action et contexte externe.
4. Lire `orchestration/` pour coordonner les composants.
5. Lire `assistant-dev/` comme étude de cas intégrée.
6. Lire `labs/` pour valider les patterns par expériences.
7. Lire `prospective/` en dernier, comme espace d'hypothèses.

- [ ] Déclic de compréhension identifié.

- [ ] Questions de conception explicites.
- [ ] Concepts reliés au glossaire si nécessaire.
- [ ] Responsabilités ou limites clarifiées.
- [ ] Compromis d'architecture visibles.
- [ ] Chapitre lisible seul.
